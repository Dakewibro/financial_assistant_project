# Design Trade-offs

1. **Manual entry vs bank sync**
   - Manual entry improves privacy and transparency.
   - Bank sync improves convenience but increases integration complexity.

2. **Few categories vs many categories**
   - Fewer categories simplify onboarding and reduce miscategorizations.
   - More categories improve analytics precision but increase cognitive load.

3. **Daily vs weekly/monthly budgets**
   - Daily limits provide immediate feedback.
   - Weekly/monthly limits better handle irregular spending.

4. **Local files vs cloud persistence**
   - Local files are easy to reason about and demo offline.
   - Cloud persistence improves collaboration and portability.

5. **Rule-based alerts vs predictive forecasting**
   - Rule-based alerts are explainable and easy to validate.
   - Predictive approaches can be smarter but harder to justify and test.

6. **MongoDB flexibility vs relational rigidity**
   - MongoDB supports faster schema iteration for student projects.
   - Relational models can enforce stronger constraints by default.
